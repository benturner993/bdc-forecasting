## Model Architecture

The production model uses a **Gradient Boosting Regressor** (scikit-learn) to predict
monthly practice revenue. Key inputs:

| Feature | Description |
|---|---|
| `nhs_mix` | NHS proportion of appointments (strong predictor of revenue per appointment) |
| `booking_rate_vs_aop` | Current-period booking rate vs AOP target |
| `bookings_lag1`, `bookings_lag3` | 1-month and 3-month lagged booking volumes |
| `rolling_avg_3m` | 3-month rolling average bookings |
| `seasonality_factor` | UK dental seasonality index by month |
| `monthly_capacity` | Total available appointment slots |
| `target_utilization` | Capacity fill target |
| `booking_profile_encoded` | Early / mixed / late booking tendency |
| `connected_care_encoded` | Connected Care area flag |

**Target**: `monthly_revenue` (ВЈ)

Confidence intervals are estimated from the standard deviation of training residuals:
- **80% CI**: В± 1.28 Г— Пѓ
- **95% CI**: В± 1.96 Г— Пѓ

## Training

```bash
# 1. Ensure real data is in data/ folder
# 2. Install dependencies
pip install scikit-learn pandas numpy

# 3. Train
python model/train_model.py --data-dir ./data --output-dir ./model
```

This will:
1. Load and merge `practices.csv`, `bookings_historical.csv`
2. Engineer lag features and encode categoricals
3. Train/test split (last ~15% of time periods = test)
4. Train a `GradientBoostingRegressor` pipeline
5. Print test MAE, RMSE, RВІ
6. Write `model.pkl` and `predictions.csv`

## Extending the Model

Consider these enhancements once real data is available:

- **Per-practice models** вЂ” train a separate model per area or booking profile
- **LSTM / temporal model** вЂ” capture longer-range booking trend signals
- **Causal factors** вЂ” incorporate marketing spend, NHS contract changes, staffing data
- **Conformal prediction** вЂ” replace Gaussian CIs with distribution-free coverage guarantees
- **Online updating** вЂ” retrain monthly as new actuals arrive
