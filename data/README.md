# Data Folder

This folder contains the datasets used by the BDC Forecasting application.
The files here are **synthetic** — generated by `data_generator.py` with a fixed random seed.
Replace them with your real data exports when ready.

For a full engineering handoff specification (metadata, schema contract, keys, and quality rules), see:
`data/DATA_SPEC.md`.

---

## Files

| File | Rows | Description |
|---|---|---|
| `practices.csv` | 380 | Practice master data — one row per practice |
| `bookings_historical.csv` | 4,180 | Monthly booking actuals (Apr 25 – Feb 26) — one row per practice × month |
| `bookings_current_month.csv` | 380 | Current month (Mar 26) partial actuals and forecasts — one row per practice |
| `aop_targets.csv` | 9,120 | Monthly AOP targets for all periods (historic + forward) — one row per practice × month |

---

## Column Reference

### `practices.csv`

| Column | Type | Description |
|---|---|---|
| `id` | string | Practice identifier (e.g. `P001`) |
| `name` | string | Full practice name |
| `location` | string | Town / neighbourhood |
| `area_id` | string | Area code (e.g. `A01`) |
| `area_name` | string | Area name (e.g. `London East`) |
| `region` | string | Geographic region |
| `nhs_mix` | float (0–1) | Proportion of appointments that are NHS |
| `private_mix` | float (0–1) | Proportion that are private |
| `avg_nhs_value` | float | Average revenue per NHS appointment (£) |
| `avg_private_value` | float | Average revenue per private appointment (£) |
| `blended_value` | float | Weighted average revenue per appointment (£) |
| `monthly_capacity` | int | Total appointment slots available per month |
| `target_utilization` | float (0–1) | Target fill rate for capacity |
| `booking_profile` | string | `early` / `mixed` / `late` — how far in advance patients book |
| `network_program` | bool | Whether the practice is in the network programme group |
| `aop_monthly_bookings` | int | AOP monthly bookings target |
| `aop_monthly_revenue` | float | AOP monthly revenue target (£) |
| `aop_annual_revenue` | float | AOP annual revenue target (£) |

### `bookings_historical.csv`

| Column | Type | Description |
|---|---|---|
| `practice_id` | string | Links to `practices.csv` |
| `year` / `month` | int | Period |
| `bookings` | int | Total appointments delivered |
| `nhs_bookings` | int | NHS appointments |
| `private_bookings` | int | Private appointments |
| `nhs_mix` | float | NHS proportion for the month |
| `revenue` | float | Total revenue (£) |
| `aop_bookings` | int | AOP bookings target for the month |
| `aop_revenue` | float | AOP revenue target for the month (£) |

### `bookings_current_month.csv`

| Column | Type | Description |
|---|---|---|
| `current_month_bookings` | int | Bookings in diary as of `day_of_month` |
| `expected_bookings_by_now` | float | Expected bookings based on historical curve × AOP target |
| `booking_rate` | float | `current_month_bookings / expected_bookings_by_now` |
| `forecast_month_total` | int | Projected end-of-month bookings |
| `forecast_month_revenue` | float | Projected end-of-month revenue (£) |
| `ci_lower` / `ci_upper` | int | 95% confidence interval bounds for end-of-month bookings |
| `within_ci` | bool | Whether AOP target falls within the CI |
| `whitespace` | float | Fraction of monthly capacity projected to be unused |
| `traffic_light` | string | `green` / `amber` / `red` (percentile-based) |

### `aop_targets.csv`

Monthly AOP targets for every practice × period combination. Covers all historical months,
the current month, and the next 12 forecast months.

## Replacing with Real Data

1. Export your source data to match the column schemas above.
2. Drop the files into this folder (same filenames).
3. Update `app.py` — replace the call to `generate_all_data()` with a CSV-loading function.
   A skeleton loader is available in `data/load_data.py` (to be created when migrating).
4. Ensure `practice_id` values are consistent across all files.
5. Recalculate `booking_rate`, `traffic_light`, and `forecast_*` fields server-side,
   or pre-compute them in your ETL pipeline.
